COMPLEXITY ROUTER — Three-Way Gate (Deterministic, No LLM)

The Complexity Router is implemented entirely in code (src/complexity-router.ts + src/proxy-server.ts).
It does NOT call the LLM. This file is documentation only.

## Three Paths

### Path 1: SCAFFOLD (Free, Instant, Deterministic)
Trigger: Issue description contains structured entity/field definitions matching CRUD patterns.
Detection: detectScaffoldConfig() in src/scaffold-engine.ts
Action: Generates all files deterministically (schema, service, routes, migration, patches shared files).
         Then sends to Local Builder for build verification only (no code generation).
         On build success → Reviewer → Diff Guardian → Visual Reviewer → PR.
Cost: Zero LLM tokens.

### Path 2: LOCAL (Cheap, Local LLM)
Trigger: Complexity score < 7 and no scaffold match.
Detection: scoreComplexity() in src/complexity-router.ts
Action: Routes directly to Strategist for decomposition via local Ollama models.
Cost: Local compute only.

### Path 3: REMOTE (Expensive, GLM-5 via z.ai)
Trigger: Complexity score >= 7 (greenfield, multi-module, architecture-level).
Detection: scoreComplexity() in src/complexity-router.ts
Action: Calls GLM-5 Remote Architect for architecture spec, then routes to Strategist with spec attached.
Cost: ~$0.01-0.05 per call (z.ai pricing).

## Complexity Scoring (0-10)
Baseline: 3

Increases:
+2: "from scratch", "greenfield", "whole app"
+2: "multi-module", "full stack", "end to end"
+2: "architect", "system design", "infrastructure"
+2: "build a complete/full/entire/whole [app/system]"
+2: [Goal] or [Epic] tag in title
+1: auth, oauth, jwt, session
+1: realtime, websocket, streaming
+1: migration, schema overhaul
+1: multiple services/endpoints/modules

Decreases:
-2: fix, bug, broken, crash, error, typo
-1: update text/label/color/style
-1: add a button/field/input
-1: rename, move, refactor
-1: cosmetic, spacing, padding
-1: crud, simple api, basic endpoint

## Scaffold-able Ticket Format
For the scaffold engine to detect and generate code, tickets should include:
- Entity name (kebab-case): entity: payment-types
- Table name (PascalCase): table: PaymentTypes
- Fields list with types:
  - name: text
  - amount: integer
  - shopId: text
- CRUD signals in title/description

## ORG CHART
- Complexity Router is the first agent to see every new issue
- Outputs to: Scaffold Engine (path 1), Strategist (path 2), Remote Architect → Strategist (path 3)
